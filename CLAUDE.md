# Beta Ads — Design System & Rules

## Stack
React + TypeScript + Vite SPA. Tailwind CSS v3 with CSS custom properties. shadcn/ui (new-york). framer-motion for animations. lucide-react for icons.

## Brand
- Primary (signal red): `#e94f37` / `hsl(357 70% 60%)`
- Dark background: `hsl(240 11% 5%)`
- Light background: white
- Serif accent font: `'Instrument Serif', serif` — italic only, for hero heading accents

---

## Design Philosophy: Apple-level Quality

Every page should feel like it was designed by Apple's marketing team — not generated by AI.
**The test:** Would this look at home on apple.com?

### What that means in practice

**Typography does the heavy lifting.** Large, confident headings with generous whitespace between sections. No need to put borders around everything to create separation — whitespace, type scale, and subtle dividers do that.

**Restraint is the skill.** One primary color accent per section maximum. No rainbow of brand colors. Not every item needs an icon. Not every section needs a card border.

**Specificity over generics.** Write actual numbers, real platform names, concrete claims. Never "powerful features" or "seamless experience" or "cutting-edge solution". Every word earns its place.

**No AI slop patterns:**
- ❌ 3-column grid of icon + bold title + 2-line description
- ❌ "Here's how it works" with 3 equally-spaced numbered steps and stock icons
- ❌ Generic testimonial carousels with star ratings
- ❌ Gradient blobs as decoration
- ❌ "Why choose us?" sections with checkmark lists
- ❌ Hero with large heading, short paragraph, and two CTA buttons centered on the page (unless it's genuinely the right layout)
- ❌ Every stat wrapped in a colored card

## Typography Rules

### Scale
| Element | Classes |
|---------|---------|
| Hero H1 | `text-5xl md:text-6xl lg:text-7xl font-bold tracking-tight leading-[1.05]` |
| Section H2 | `text-3xl md:text-4xl font-light tracking-tight` |
| Card H3 | `text-base font-semibold` |
| Section label | `text-xs font-semibold tracking-widest uppercase text-primary` |
| Body | `text-sm leading-relaxed text-muted-foreground` |
| Large body | `text-base md:text-lg font-light leading-relaxed` |
| Stat value | `text-2xl font-bold tracking-tight` |
| Stat label | `text-xs text-muted-foreground` |

### Rules
- Section headings: always `font-light tracking-tight`. Not bold.
- Section labels (category markers above headings): `text-xs font-semibold tracking-widest uppercase text-primary mb-3 block`
- Hero headings: one line in normal weight, can have italic serif accent word via `style={{ fontFamily: "'Instrument Serif', serif" }} italic font-normal`
- Never center body text except in isolated CTA blocks
- Body copy lives in `max-w-2xl` or `max-w-xl` columns — never full width

## Layout Rules

### Container
```
max-w-7xl mx-auto px-6 lg:px-12
```

### Section Padding
```
py-20 md:py-28
```
Use `border-t border-border` to separate sections (not extra padding or dividers).

### Hero
```
pt-36 pb-28
```
Content max-width: `max-w-2xl`

### Two-column layouts
```
grid lg:grid-cols-2 gap-16 items-start
```
Left: heading + body copy. Right: feature items or visual. Never symmetric — left should feel heavier.

## Component Patterns

### Hero Section
```tsx
<section className="relative overflow-hidden">
  <AnimatedShaderBackground heightFactor={0.85} />
  <div className="absolute inset-x-0 bottom-0 h-48 bg-gradient-to-t from-background to-transparent z-[1] pointer-events-none" />
  <div className="relative z-10 max-w-7xl mx-auto px-6 lg:px-12 pt-36 pb-28">
    <div className="max-w-2xl">
      {/* Platform badge */}
      <span className="inline-flex items-center gap-2 text-xs font-semibold px-3 py-1 rounded-full bg-white/10 text-white/80 backdrop-blur-sm mb-7">
        <img src="..." className="h-3.5 w-auto" />
        Page Title
      </span>
      {/* H1 with optional serif accent */}
      <h1 className="text-5xl md:text-6xl lg:text-7xl font-bold text-white leading-[1.05] mb-6 tracking-tight">
        <span style={{ fontFamily: "'Instrument Serif', serif" }} className="italic font-normal">Accent word</span>
        <br />Rest of heading
      </h1>
      {/* Subheading */}
      <p className="text-lg text-white/65 leading-relaxed mb-10 max-w-lg">...</p>
      {/* Buttons */}
      <div className="flex flex-wrap gap-3">
        <Button size="lg" className="bg-primary hover:bg-primary/90 text-white rounded-full px-8 h-12">
          Primary CTA <ArrowRight className="ml-2 w-4 h-4" />
        </Button>
        <Button size="lg" variant="ghost" className="text-white hover:bg-white/10 rounded-full px-8 h-12 border border-white/20">
          Secondary CTA
        </Button>
      </div>
    </div>
    {/* Stat strip */}
    <div className="grid grid-cols-2 md:grid-cols-4 gap-px mt-20 border border-white/10 rounded-2xl overflow-hidden bg-white/10">
      {stats.map((s) => (
        <div key={s.label} className="bg-black/30 backdrop-blur-sm px-6 py-5">
          <div className="text-2xl font-bold text-white tracking-tight">{s.value}</div>
          <div className="text-xs text-white/50 mt-0.5">{s.label}</div>
        </div>
      ))}
    </div>
  </div>
</section>
```

```tsx

  <div>
    <h2 className="text-3xl md:text-4xl font-light tracking-tight mb-3">{heading}</h2>
    <p className="text-background/60 max-w-md">{subtext}</p>
  </div>
  <div className="flex flex-wrap gap-3 shrink-0">
    <Button size="lg" className="bg-primary hover:bg-primary/90 text-white rounded-full px-8">
```

### FAQ
Use native `<details>` elements. No Accordion component unless already on the page.
```tsx
<details className="group border-b border-border last:border-b-0">
  <summary className="flex items-center justify-between py-5 cursor-pointer text-sm font-medium text-foreground hover:text-primary transition-colors select-none">
    {question}
    <ArrowRight className="w-4 h-4 shrink-0 ml-4 transition-transform group-open:rotate-90 text-muted-foreground" />
  </summary>
  <div className="pb-5 text-sm text-muted-foreground leading-relaxed">{answer}</div>
</details>
```

### Process / Steps
```tsx
<div className="grid md:grid-cols-4 gap-8">
  {steps.map((s, i) => (
    <div key={s.step} className="relative">
      {i < steps.length - 1 && <div className="hidden md:block absolute top-4 left-full w-full h-px bg-border -translate-x-4" />}
      <div className="text-4xl font-bold text-primary/15 mb-4 tracking-tighter">{s.step}</div>
      <h3 className="text-base font-semibold text-foreground mb-2">{s.title}</h3>
      <p className="text-sm text-muted-foreground leading-relaxed">{s.desc}</p>
    </div>
  ))}
</div>
```

### Scroll Animation
```tsx
const { ref, isVisible } = useScrollAnimation();
<div ref={ref} className={`transition-all duration-700 ${isVisible ? "opacity-100 translate-y-0" : "opacity-0 translate-y-8"}`}>
```
Only apply scroll animations to content that starts **below the fold**. Above-fold content must be visible immediately.

## Buttons

All CTAs on pages: `rounded-full`. Never `rounded-md` for primary page buttons.

```tsx
// Primary
<Button size="lg" className="bg-primary hover:bg-primary/90 text-white rounded-full px-8 h-12">
  Book a Demo <ArrowRight className="ml-2 w-4 h-4" />
</Button>

// Secondary (on dark/hero)
<Button size="lg" variant="ghost" className="text-white hover:bg-white/10 rounded-full px-8 h-12 border border-white/20">
  Learn More
</Button>

// Secondary (on light/CTA block)
<Button size="lg" variant="ghost" className="text-background hover:bg-background/10 rounded-full px-8 border border-background/20">
  View Pricing
</Button>
```

## Page Structure

Every marketing page follows this structure:
```
<MarketingPageLayout seo={...} cta={{ heading, subtext, primaryBtn, secondaryBtn }}>
  {/* Hero section is NOT inside layout — it's the first child, uses full viewport */}
  <HeroSection />

  {/* Content sections: each gets py-20 md:py-28 + border-t border-border */}
  <ContentSection />
  <ContentSection />

  {/* CTA section is provided via layout prop — auto-rendered before footer */}
</MarketingPageLayout>
```

Use `<MarketingPageLayout>` from `@/components/layout/MarketingPageLayout` for all new marketing/platform pages.

## What NOT to Do

1. **No centered hero text** — left-aligned headings feel more confident and Apple-like
2. **No generic stock photo backgrounds** — use AnimatedShaderBackground or solid backgrounds
3. **No emoji in UI** unless the user explicitly asked for it
4. **No "✓ Feature" lists inside cards** — if you need a checklist, use a clean list with `text-sm` items and a small check icon
5. **No boxed stat cards with colored backgrounds** — use the borderless stat strip pattern instead
6. **No `dark:invert` on logo images** — it inverts brand colors incorrectly
7. **No multiple font weights in the same heading** — pick one weight per heading (bold for h1, light for h2)
8. **No gratuitous backdrop-blur** — use it only in the hero overlay and floating controls
9. **Don't use `max-w-7xl` for text columns** — body copy always lives in `max-w-2xl` or narrower
10. **Don't add loading spinners** for data that loads instantly
